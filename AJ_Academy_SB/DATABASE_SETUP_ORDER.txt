AJ Academy — Supabase SQL setup order
=====================================
Run in Supabase SQL Editor (same project as NEXT_PUBLIC_SUPABASE_URL).

1) schema.sql
   Profiles, employee_details, audit_logs, helpers (get_user_role, is_admin).
   Roles: super_admin, admin, student, freelancer, mentor.

2) attendance_module.sql
   Attendance tables + RLS (includes check_in_selfie_url column).

3) attendance_selfie_schema.sql
   Storage bucket attendance-selfies for freelancer check-in photos.

4) task_schema.sql
   Tasks table + base RLS.

5) aj_academy_roles_patch.sql
   Task policies for student / freelancer / mentor assignees.

6) task_notifications_columns.sql
7) in_app_notifications.sql
8) profiles_rls_fix.sql   *** REQUIRED for login (RLS + get_my_profile RPC) ***
8b) add_role_and_sync_profiles.sql  *** after Auth users exist — fills role column + links Auth → profiles ***
8c) add_employee_role.sql  *** run if User Master fails with profiles_role_check for employee role ***
9) project_master_schema.sql (projects + tasks.project_id; includes minimal clients stub)
10) company_policies_schema.sql (optional)
10b) company_policies_category.sql (employee + freelancer policy tabs)
10c) task_assignee_snapshot.sql (freelancer offboard — keep completed tasks)
10d) mentor_department_tasks.sql (mentor sees students in same department)
10d2) freelancer_department_tasks.sql (freelancer assign tasks — same department students)
10f) aj_academy_platform_expansion.sql (course, mentor assign, task attachments, counselling, notifications)
10f3) task_completion_attachments.sql (student task completion files for assigner)
10f7) reimbursement_member_patch.sql (draft claims, bill uploads — after reimbursement_schema_patch.sql)
10f2) counselling_student_contact_schema.sql (legacy — included in patch + expansion)
10g) employee_module_schema.sql (permission columns + leave_requests for employee panel)
10h) employee_daily_mood_schema.sql (optional — mood survey on employee login)
10i) employee_lead_management_schema.sql (employee Lead Management — adds assigned_to on clients if missing)
10e) system_settings_rls_fix.sql (admin can save departments / settings — run if RLS error on delete)
11) attendance_delete_grants.sql (optional — admin delete attendance)
11b) remove_leave_requests_module.sql (optional — drops leave_requests; use permission_requests only)

DO NOT run rls-policies.sql — it is legacy Birthmark Brahma (employee/manager) and breaks AJ Academy login.

Skip legacy BB-only scripts unless you need CRM/finance / Student Master:
  student_lead_master_schema.sql, student_lead_master_aux_schema.sql,
  student_lead_master_columns_patch.sql (if 42703 column missing),
  student_master_columns_patch.sql (Student Master counselling fields — after student_lead_master_schema.sql),
  student_lead_master_rls_fix.sql (RLS hardening — after schema + employee_lead_management_schema.sql),
  finance_schema.sql, employee_daily_mood_schema.sql, etc.

After SQL: create users in Authentication, then insert profiles rows (see SUPABASE_SETUP_GUIDE.md).

Optional email notifications:
  Set RESEND_API_KEY (+ TASK_EMAIL_FROM) in AJ_Academy_OS env to send counselling schedule emails.

Student portal (/student/*):
  Same SQL as employee panel for attendance, permission, leave, policies, profile.
  Also run aj_academy_platform_expansion.sql + counselling_sessions_patch.sql for course/mentor + counselling history.

Freelancer portal (/freelancer/*):
  Uses task_schema.sql + freelancer_department_tasks.sql for assigner flow.
  My Tasks (tasks assigned to freelancer) + popup notifications use in_app_notifications.sql.
  Mood popup requires optional employee_daily_mood_schema.sql.

Mentor portal (/mentor/*):
  attendance_module.sql + mentor_department_tasks.sql + aj_academy_platform_expansion.sql.
  My Tasks (assignee) + Assign Tasks (assigner) + student roster on dashboard.
  Mood popup optional via employee_daily_mood_schema.sql.
